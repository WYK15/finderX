# FinderX Project Rules

## UI Design Baseline

Before adding or changing FinderX pages, dialogs, settings panels, toolbar actions, sidebars, file lists, path bars, context menus, drag feedback, preview surfaces, or icons, read [Design.md](Design.md).

Use `Design.md` as the visual and interaction baseline. New UI should follow the SpaceUI-inspired FinderX direction documented there: compact file-manager ergonomics, semantic theme tokens, quiet surfaces, icon-first controls, clear selection/focus states, and consistent icon sourcing.

When replacing or adding icons, check the icon strategy in `Design.md` first. SpaceUI's icon package can be used as a style and asset reference, but directly imported assets must keep the required MIT license attribution.
